# Домашнее задание №4 «Публикация и развитие проекта»

### Работа с плагинами

Подключите плагин mkdocs-glightbox по [инструкции](../pet-project/mkdocs-plugins.md#mkdocs-glightbox).

### Публикация проекта

Для выполнения домашнего задания используйте инструкцию [Публикация проекта](../pet-project/mkdocs-publication.md).

1. Опубликуйте проект на GitHub Pages с помощью команды `mkdocs gh-deploy`. 
2. Настройте СI/CD для автоматической публикации проекта с помощью GitHub Pages.

### Разработка документации

1. Познакомьтесь с документацией приложения [Нотсапп](../pet-project/notesapp/specification.md)
2. Подготовьте описание одного из пользовательских сценариев для приложения Нотсапп. Используйте пример: [Пользовательский сценарий «Создание новой заметки»](../pet-project/notesapp/scenario-1.md)
3. Разместите описание сценария в свой домашний проект и опубликуйте изменения.

### Проверка домашнего задания

Пришлите в чат курса ссылку на пользовательский сценарий, опубликованный на вашем домашнем проекте.
