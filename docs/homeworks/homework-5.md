# Домашнее задание №4 «Добавление разных типов документации»

### Спецификация API

1. Скачайте файл спецификации [openapi.yml](../openapi.yml){target=_blank} к себе на устройство.
2. Добавьте файл спецификации в свой домашний проект.
3. Установите плагин `swagger-ui-tag` по [инструкции](../pet-project/api-add.md)
4. Создайте статью для публикации спецификации и добавьте в нее тег со ссылкой на файл спецификации по [инструкции](../pet-project/api-add.md).
5. Добавьте статью в конфиг.
6. Обновите worker-файл с учётом подключения нового конфига.
7. Пересоберите проект.

### Диаграмма последовательности PlantUML

#### Подключение плагина

1. Установите и подключите плагин `mkdocs_puml` по [инструкции](../pet-project/plant-uml.md#plugin).
2. Обновите worker-файл с учётом подключения нового конфига.

#### Разработка диаграммы

1. Дополните ваш пользовательский сценарий API-запросами из подключённой спецификации.
2. Разработайте диаграмму последовательности для пользовательского сценария по [инструкции](../pet-project/plant-uml.md#sequence-diagram)
3. Добавьте диаграмму на страницу с пользовательским сценарием.
4. Проверьте, что диаграмма генерируется.
5. Пересоберите проект.

### Проверка домашнего задания

Пришлите в чат курса ссылки на API-спецификацию и пользовательский сценарий с диаграммой.
