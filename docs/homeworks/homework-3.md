# Домашнее задание №3 «Создание и базовая настройка проекта»

Для выполнения домашнего задания используйте инструкцию [Создание и базовая настройка проекта](../pet-project/mkdocs-setup.md).

1. Установите **MkDocs** и **Material for MkDocs** с помощью пакетного менеджера Python (pip).
2. Создайте локальный репозиторий.
3. В рабочей директории локального репозитория создайте новый проект MkDocs.
4. Добавьте текст на главную страницу проекта (`docs/index.md`)
5. Настройте конфиг `mkdocs.yml`.

    Поэкспериментируйте с настройками в конфиге. Проверить отображение настроек вы можете с помощью локальной сборки.
6. Добавьте в проект инструкцию из домашнего задания №1, укажите её в разделе навигации конфига.

    !!! note "Примечание"
        Практическая работа по кросс-ревью проходит параллельно и независимо от разработки домашнего проекта, поэтому в домашний проект вы можете добавить первую версию вашей инструкции, ещё не прошедшую ревью.

7. Соберите проект локально, проверьте, что всё отображается корректно.
8. Создайте удалённый репозиторий проекта в GitHub и свяжите его с локальным репозиторием.
9. Закоммитьте изменения в локальном проекте и опубликуйте ветку `master` в удалённом репозитории.

### Дополнительное занятие

Если у вас есть дополнительные материалы для публикации в домашнем проекте (тестовые задания, документы из портфолио), переведите их в markdown и добавьте в проект.

### Проверка домашнего задания

Пришлите в чат курса ссылку на проект в GitHub.
