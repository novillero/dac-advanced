1. Docs as code как симбиоз практик и инструментов.
2. База знаний как продукт.
3. Унификация документов и текста.
4. Как мы контролируем изменения и управляем ими.
5. Ревью как часть процесса документирования.
6. Генерация базы знаний.
7. Автоматизация: непрерывная интеграция и доставка текстов пользователю.
8. Жизненный цикл документа.
9. Цели и преимущества docs as code. Почему docs as code — не панацея.
10. Как мы будем заниматься: цели и задачи курса.
