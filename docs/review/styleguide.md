# Стайлгайд. Пошаговые инструкции

#### 1. Заголовки

1.1 Статья должна начинаться с заголовка первого уровня:

```markdown title="Пример"
# Заголовок статьи
```

1.2 В статье может быть только один заголовок первого уровня.

1.3 В конце заголовка не ставится точка.

#### 2. Текст

2.1 Абзацы текста должны отделяться друг от друга и от остальных элементов форматирования пустой строкой:

```markdown title="Пример"
Первый абзац

## Заголовок

Второй абзац
```

#### 3. Списки

3.1 Пункты списка должны отделяться друг от друга и от остального текста пустой строкой:

```markdown title="Пример"
1. Первый пункт.

2. Второй пункт.

Остальной текст.
```

3.2 Перед вложенным пунктом списка необходим отступ в четыре пробела относительно родительского пункта:

```markdown title="Пример"
1. Первый пункт:

    * Вложенный пункт 2 уровня.

        * Вложенный пункт 3 уровня.

```

3.3 В пункты списка также можно вкладывать текст и изображения:

```markdown title="Пример"
1. Первый пункт:

    ![](./image.png)
    
    Вложенный текст
```

3.4 В конце пунктов списка должны ставиться точки:

```markdown title="Пример"
1. Первый пункт.

2. Второй пункт.
```

3.5 Если у пункта есть вложенное содержимое (включая вложенные подпункты), в конце пункта должно ставиться двоеточие:

```markdown title="Пример"
1. Первый пункт:

    * Вложенный пункт.
```

#### 4. Элементы интерфейса

4.1 Названия элементов интерфейса должны писаться с большой буквы.

4.2 Названия кнопок должны выделяться жирным начертанием:

```markdown title="Пример"
1. Нажмите кнопку **Отправить**.
```

4.3 Названия полей должны выделяться кавычками-ёлочками:

```markdown title="Пример"
1. Заполните поле «Описание канала»
```

4.4 В тексте допустимо использовать следующие варианты названия приложения:

* Telegram.

* телеграм.

#### 5. Стилистика

5.1 Действия инструкции, которые выполняет пользователь, должны описываться с помощью глаголов в повелительном наклонении:

```markdown title="Пример"
Откройте приложение, перейдите на вкладку.
```

5.2 Слово «пожалуйста» не используется при описании пошаговых действий пользователя.
