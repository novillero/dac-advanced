# Установка плагинов для MkDocs

Плагины MkDocs — это дополнительные модули, которые реализуют новые функциональности стандартного MkDocs.

## Встроенные плагины

Встроенные плагины не нужно устанавливать, они работают из коробки. Достаточно подключить их в конфиге проекта.

### Search

Search — встроенный плагин поиска по сайту. Плагин добавляет в шапку сайта окно поиска, индексирует статьи и генерирует json-файл с данными для поиска, который загружается в кэш-память браузера пользователя. Весь процесс поиска происходит на стороне пользователя.

Для подключения активируйте плагин в конфиге:

```yaml title="mkdocs.yml"
plugins:
  - search
```

Документация плагина: [https://squidfunk.github.io/mkdocs-material/setup/setting-up-site-search/](https://squidfunk.github.io/mkdocs-material/setup/setting-up-site-search/){target=_blank}

### Tags

Tags — встроенный плагин для создания системы тегов. Теги позволяют категоризировать статьи и группировать их по общим признакам.

Для подключения активируйте плагин в конфиге:

```yaml title="mkdocs.yml"
plugins:
  - search
  - tags
```

Документация плагина: [https://squidfunk.github.io/mkdocs-material/setup/setting-up-tags/](https://squidfunk.github.io/mkdocs-material/setup/setting-up-tags/){target=_blank}

#### Настройка тегов

1. Подумайте, на какие категории вы хотите разделить ваши статьи, и составьте список тегов для этих категорий.
2. Добавьте список тегов в конфиг:

    ```yaml title="mkdocs.yml"
    plugins:
      - search
      - tags:
          tags_allowed:
          - tag1
          - tag2
          - tag3
    ```

3. Добавьте теги на страницы. Вставьте разметку в начало страницы (над заголовком первого уровня):

    ```markdown
    ---
    tags:
      - tag1
    ---
    # Заголовок статьи
    ```

    !!! note "Примечание"
        Вы можете отметить статью сразу несколькими тегами:
        
        ```markdown
        ---
        tags:
          - tag1
          - tag2
          - tag3
        ---
        ---
        # Заголовок статьи
        ```

#### Создание теговых страниц

Плагин создаёт списки статей, группируя их по тегам.

Чтобы добавить такой список на страницу, используйте разметку:

!!! quote ""
    &lt;!-- material/tags --&gt;

На страницу добавится список всех тегов и статей, отмеченных этими тегами.

Чтобы добавить на страницу список определённых тегов, используйте разметку:

!!! quote ""
    &lt;!-- material/tags { include: [tag1, tag2] } --&gt;

### Пример    

1. В рамках документации по курсу «Docs as Code» все статьи поделены на категории:

    |Категория            |Тег      |
    |:-------------------:|:-------:|
    |Задания              |task     |
    |Справочные материалы |help     |
    |Руководства          |manual   |
    |Работа с GIT         |git      |
    |Домашний проект      |pet      |
    |Маркдаун             |markdown |

2. В конфиге `mkdocs.yml` активирован плагин `tags` и добавлен список тегов:

    ```yaml title="mkdocs.yml"
    plugins:
      - search
      - tags:
          tags_allowed:
          - task 
          - help 
          - manual
          - git
          - pet  
          - markdown
    ```

3. На все статьи добавлены нужные теги:

    ```
    ---
    tags:
      - help
      - pet
    ---

    # Установка плагинов для MkDocs
    ```

4. На [теговую страницу](../tags.md) выведен список всех тегов.

## Внешние плагины

Внешние плагины необходимо предварительно установить с помощью Python installs Packages (pip), а потом активировать в конфиге проекта.

### MkDocs GLightbox

Плагин создаёт масштабируемые галереи изображений из добавленных в статью проекта картинок.

1. Установите Glightbox с помощью pip:

    ```
    pip install mkdocs-glightbox
    ```

2. Активируйте плагин в конфиге проекта:

    ```yaml title="mkdocs.yml"
    plugins:
      - search
      - glightbox
    ```

Ссылка на плагин: [https://github.com/blueswen/mkdocs-glightbox](https://github.com/blueswen/mkdocs-glightbox){target=_blank}
